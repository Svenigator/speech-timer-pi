Features

Alles was du brauchst

Speech Timer Pi deckt alles ab – vom einfachen Countdown bis zur professionellen Event-Technik mit Stream Deck.

📺

Vollbild-Display

Große, klar lesbare Ziffern auf dem Pi-Monitor oder LED-Wand. Kiosk-Modus startet automatisch nach dem Boot.

🌐

Web-Steuerung

Steuerung von jedem Gerät im Netzwerk über den Browser. Keine App, kein Install – einfach IP aufrufen.

🏠

Warnzonen

Konfigurierbare Farbphasen: grün → gelb bei 3 Min → rot bei 30 Sek → blinkend bei Overtime mit negativer Zeit.

📋

Presets

Vorkonfigurierte Zeiten für verschiedene Vortragsformate. Laden und separat starten – für präzises Timing.

⚙

Live-Anpassung

+/- 1 Min und +/- 5 Min auch während der Timer läuft. Kein Stoppen notwendig.

🕐

Echtzeit-Uhr

Display umschaltbar zwischen Timer-Countdown und aktueller Uhrzeit. Zeitzonen-Einstellung direkt im Browser.

📶

OSC-Integration

Open Sound Control für Steuerung und Status-Updates. Perfekt für bestehende AV-Setups und Lichttechnik.

📡

WLAN-Manager

WLAN-Netzwerke scannen und verbinden direkt aus dem Browser. Kein SSH für Netzwerk-Setup nötig.

💻

Systemd-Autostart

Startet automatisch bei jedem Boot. Kein Login, kein manuelles Starten – einfach einschalten und loslegen.

Quick Start

In 4 Schritten installiert

Voraussetzung: Raspberry Pi 4 oder 5 mit Raspberry Pi OS Bookworm (64-bit Desktop).

1

Repository klonen

Das Repo auf den Pi klonen und in das Verzeichnis wechseln.

# Repository herunterladen
git clone https://github.com/Svenigator/speech-timer-pi.git
cd speech-timer-pi
2

Dateien an die richtige Stelle kopieren

Die Pi-App-Inhalte in das Home-Verzeichnis kopieren.

mkdir -p ~/speech-timer-pi
cp -r pi-app/. ~/speech-timer-pi/
3

Install-Script ausführen

Das Script installiert alle Abhängigkeiten, richtet Systemd ein und konfiguriert den Kiosk-Modus.

cd ~/speech-timer-pi
bash scripts/install.sh
4

Neustart

Nach dem Neustart startet der Browser automatisch im Kiosk-Modus. Web-Steuerung erreichbar unter:

sudo reboot

# Danach erreichbar unter:
http://<Pi-IP>/control
Hardware

Empfohlenes Setup

Speech Timer Pi ist optimiert für Raspberry Pi 4 und 5. Beide Modelle werden vollständig unterstützt.

📌 Raspberry Pi 5 (Empfohlen)

  • 5V/5A USB-C-Netzteil (offizielles RPi-Netzteil)
  • Mini-HDMI Kabel (nicht Micro-HDMI wie Pi 4)
  • HDMI0 = Port näher am USB-C-Anschluss
  • Raspberry Pi OS Bookworm 64-bit (Desktop)
  • Ethernet oder WLAN

📌 Raspberry Pi 4

  • Micro-HDMI Kabel
  • 5V/3A USB-C-Netzteil
  • Raspberry Pi OS Bookworm 64-bit (Desktop)
  • Ethernet oder WLAN

📺 Ausgabe

  • HDMI-Monitor (beliebige Auflösung)
  • LED-Wand mit HDMI-Eingang
  • Projektor (für grosse Räume)
  • Zweiter Monitor als Presenter-Sicht

📱 Steuerung

  • Smartphone/Tablet: Browser genügt
  • Stream Deck direkt per USB am Pi
  • Stream Deck via Bitfocus Companion
  • Jede OSC-fähige App oder Hardware
Integrationen

Nahtlos integriert

Speech Timer Pi spielt perfekt mit deiner bestehenden Technik zusammen.

▶

Stream Deck direkt

USB-Anschluss direkt am Pi ohne Companion. Auto-Detection des Modells (Mini, Neo, MK.2, XL). Hot-Plug im Betrieb.

🔗

Bitfocus Companion

Ready-to-use Companion-Modul inklusive. Einfach installieren, IP eintragen – alle Buttons und Feedbacks sofort verfügbar.

🎶

OSC

Open Sound Control für Steuerung und Status-Broadcasts. Kompatibel mit QLab, TouchDesigner, ETC Eos und mehr.

📷

Netzwerk-API

REST-API für eigene Integrationen. Hostname, IPs, SSID, Preset-Namen – alles abrufbar per HTTP.

SSH-Befehle

Fernzugriff & Verwaltung

Alle wichtigen SSH-Befehle für die Verwaltung des Pi und die Website im Unterordner speech-Timer-pi.

SSH-Verbindung zum Pi 
# Standard-Verbindung
ssh pi@<PI-IP-ADRESSE>

# Mit eigenem Benutzernamen
ssh <benutzername>@<PI-IP-ADRESSE>

# Pi-IP herausfinden (im Netzwerk)
arp -a | grep raspberry

# Oder direkt am Pi:
hostname -I
Website-Verzeichnis einrichten 
# Webserver-Root fuer Nginx (Standard)
sudo mkdir -p /var/www/html/speech-Timer-pi

# Repo klonen in Unterordner
cd /var/www/html
sudo git clone https://github.com/Svenigator/speech-timer-pi.git speech-Timer-pi

# Oder nur den website/ Ordner kopieren
sudo cp -r ~/speech-timer-pi/website/. \
  /var/www/html/speech-Timer-pi/
Nginx als Webserver einrichten 
# Nginx installieren
sudo apt update && sudo apt install -y nginx

# Konfigurationsdatei erstellen
sudo nano /etc/nginx/sites-available/speech-timer

# Inhalt fuer den Unterordner:
server {
  listen 80;
  root /var/www/html;
  index index.html;
  location /speech-Timer-pi {
    try_files $uri $uri/ =404;
  }
}

# Aktivieren und starten
sudo ln -s /etc/nginx/sites-available/speech-timer \
  /etc/nginx/sites-enabled/
sudo systemctl restart nginx
Service-Verwaltung 
# Timer-App Status pruefen
sudo systemctl status speech-timer.service

# Service neu starten
sudo systemctl restart speech-timer.service

# Logs in Echtzeit
journalctl -fu speech-timer.service

# Website mit Python schnell testen
cd /var/www/html/speech-Timer-pi
python3 -m http.server 8080
# Dann: http://<PI-IP>:8080

# Dateien per SCP auf den Pi kopieren
scp -r ./website/ pi@<PI-IP>:/var/www/html/speech-Timer-pi/
